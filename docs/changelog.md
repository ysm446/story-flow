# Changelog

## 未リリース

### 2026-07-10 00:43 — 修正: モデルバーの名前が二重に見える

- ヘッダーのモデルバーが「フォルダ名/ファイル名」を表示していたため、フォルダ名が
  モデル名とほぼ同じ構成（HuggingFace 慣習）だと二重に見えていた。ファイル名のみの
  表示に変更（フルパスはホバーで確認可能）

### 2026-07-10 00:34 — LLM の思考モードを確実に無効化

- 思考モード（reasoning/thinking）対応モデルでも、清書・選曲が思考なしで実行されるように
  - サーバ起動引数に強制打ち切り（--reasoning-budget 0）と Qwen3 系向けの
    enable_thinking=false を追加
  - リクエスト側でも無効化を指定（手動起動したサーバに繋いだ場合にも効く）

### 2026-07-10 00:28 — 安定性の修正（プロジェクトレビューでの検出分）

- 修正: BGM をノードに指名したあとその BGM を削除すると、生成が全シーン完了した後の
  保存で失敗し、テイクが丸ごと失われていた（無効な参照は自動選曲/無音に劣化して保存）
- 修正: 生成中に別タブへ移動すると進行が見えなくなり、二重に生成を開始できてしまっていた
  （Compose / Generate は画面を切り替えても状態を保持するように）
- 修正: Compose 編集直後（0.8 秒以内）にタブを切り替えると、その編集が保存されず消えていた
- 修正: Compose でノードを追加・削除したあと、Generate の「新しいテイクを生成」が
  編集前の古い構成で生成されることがあった（常に現在のグラフから生成する）
- 修正: 生成が完了しないまま接続が切れた場合に成功と表示されていた（エラー表示に）
- 修正: カード保存（埋め込み計算）中に作品の自動保存などが重なると
  「database is locked」で失敗することがあった
- アプリの二重起動を禁止（2 つ目は既存ウィンドウを前面に出すだけ）
- 前回の異常終了で残った llama-server を起動時に自動回収
  （残っているとサーバが増殖して VRAM を余分に消費していた）
- モデルの起動失敗（ファイル破損など）を 90 秒待たずに即エラー表示

### 2026-07-08 23:51 — フェーズ 2: Generate（逐次生成）

- アンカー列を左から 1 シーンずつ清書し、確定事実（StoryState）を持ち越す生成パイプライン
- 生成は SSE でシーン毎に届き、画面にシーンが順に埋まっていく進行表示
- プロット・目標トーン（結末の着地）の指定
- 生成前に writer モデル（llama-server）を自動起動
- 清書用 system prompt の編集機能（上書き保存 / 既定に戻す。出力形式指示は自動付与）
- 物語は stories / story_scenes に保存（Theater で再生予定）

### 2026-07-08 23:14 — フェーズ 1: Vault（素材管理）

- シーンカードの登録・編集・削除（タイトル / ブリーフ / ロール / トーン / タグ 3 種）
- 画像・動画のアップロード（sha256 命名で `data/library/` に保存、サムネイル自動生成）
- キーワード検索（FTS5）と意味検索（ベクトル）、ロール絞り込み
- 在庫密度チップ（ロール別枚数）と埋め込み済み枚数の表示
- 類似カード確認（重複検知）
- 埋め込みサーバ（Qwen3-Embedding-4B）の自動起動とセットアップパネルでの起動/停止

  進捗表示、キャンセル対応）と models/ 配下 GGUF のロード/停止 UI
- SQLite スキーマ（sqlite-vec / FTS5 含む）と Vault 在庫密度 API の初期実装
- `start.bat`（初回セットアップ + 起動）
