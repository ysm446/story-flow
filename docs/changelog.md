# Changelog

## 未リリース

### 2026-07-09 05:13 — 設定の保存先を data/settings.json に

- 環境設定（Theater/Generate/ステータスバーの各設定）を data/settings.json に保存
  （旧保存分は初回起動時に自動移行）

### 2026-07-09 05:11 — フラットアイコン + Generate ノードのサムネイル

- アプリ内のアイコンを絵文字からフラットな線画アイコンに統一
- Generate のノードにもカードのサムネイル（動画バッジ付き）を表示

### 2026-07-09 05:04 — ステータスバー + Generate ノードの表示改善

- 下部にステータスバーを追加。右端に CPU / RAM / GPU / VRAM のリソースモニタ
  （📈 アイコンで表示のオンオフ）、左に backend とモデルの状態
- Generate のノードは文章の長さに合わせて縦に伸びる表示に（内部スクロール廃止）

### 2026-07-09 04:58 — Generate のフローチャート化 + テイク（部分再生成と後戻り）

- Generate 画面がフローチャートに。Compose と同じ配置のノードに清書文が流れ込む
- 各ノードから「このシーンのみ」「ここから最後まで」の部分再生成が可能
  （直前シーンの確定事実から再開。1 シーンのみの場合、後続には「要確認」表示）
- 生成結果は毎回「テイク」として保存され、一覧からいつでも過去のテイクに戻れる
  （テイクを表示してそこから撮り直しもできる）

### 2026-07-09 04:52 — 清書時に画像を反映

- 清書時にシーンのメディア（サムネイル）を LLM に見せ、写っている情景・色・光を
  描写へ反映するように（vision 対応モデルのみ。動画はサムネイルフレームを使用）
- 設定「カードの画像を清書に反映」でオン/オフ（既定オン）

### 2026-07-09 04:41 — Generate のストリーミング表示

- 生成中のシーンの本文が、書き上がるのを待たずに 1 トークンずつ画面に流れるように

### 2026-07-09 04:37 — 表示の微調整

- 再生時の本文エリアの高さを画面の 1/4 程度に抑えた（長文は下部で見切れてスクロール）
- 設定画面のスライダーを細く（トラック 3px・つまみ 12px）

### 2026-07-09 04:31 — 再生ステージのサイズ + 全画面再生

- 設定「再生ステージのサイズ」（70〜100%）: 100% 未満は黒背景の中央に額縁表示
- 再生中の ⛶ ボタンまたは F キーで全画面再生（Esc で解除 → もう一度で一覧へ）
- 修正: 動画再生中に本文テキストが表示されなくなっていた問題

### 2026-07-09 04:27 — クロスディゾルブの暗転修正 + Theater 設定の拡充

- 動画ループのクロスディゾルブ中に画面が暗くなる問題を修正
  （下の動画を不透明のまま残し、上の動画だけをフェードインする方式に変更）
- 設定に「文字送りの速さ」（ms/字）と「本文のフォントサイズ」を追加

### 2026-07-09 04:19 — ノードの追加指示 + ロール任意化

- Compose でノードを選択すると右パネルに「ノードのプロパティ」が出て、
  その作品でのシーンへの追加指示を書ける（清書時にブリーフへ加えて反映）
- カードのロールは任意に（既定「自動」）。付けたい場合だけ選べばよい
- Compose 左のカード一覧にサムネイルとブリーフのプレビューを表示
- ウィンドウをコンテンツ領域 1920×1080 で起動

### 2026-07-09 04:02 — 動画ループのクロスディゾルブ

- Theater の動画シーンで、ループの継ぎ目を終端と先頭のクロスディゾルブで滑らかに接続
- 設定画面からオン/オフ可能（オフで通常のカットループ）
- クロスディゾルブの長さをスライダーで調整可能（0.2〜3.0 秒、既定 1.0 秒）

### 2026-07-09 01:55 — シーンの長さ設定 + 長文の再生対応

- Compose の生成設定に「シーンの長さ」（短め / 標準 / 長め）を追加。作品ごとに保存
- Theater: 画面に収まらない長文は、ストリーミング時は文字に追従してスクロール、
  オフ時は表示時間に合わせてゆっくり自動スクロール

### 2026-07-09 01:44 — Compose ノードのサムネイル + 動画のインライン再生

- Compose のノードにも動画のサムネイル（🎬 バッジ付き）を表示
- カード編集: 保存済み動画はサムネイルクリックで再生/停止（差し替えは「ファイル変更」ボタン）

### 2026-07-09 01:39 — 動画サムネイルの表示

- Vault の一覧とカード編集で、動画カードもサムネイル（ffmpeg でフレーム抽出）を表示
- 動画には 🎬 バッジを重ねて画像と区別。選択直後の動画はその場でループプレビュー

### 2026-07-09 01:26 — Vault のドラッグ&ドロップ

- カード編集のメディア枠に画像/動画をドロップで設定できるように
- Vault の一覧に直接ドロップすると、そのファイル付きで新規カード作成が開く

### 2026-07-09 01:03 — ワークスペース（作品単位の保存）

- 「作品」単位で構成（Compose のグラフ・プロット・トーン・プロンプト）を切替・保存
  できるように。編集は自動保存され、アプリを閉じても消えない
- Compose 左サイドバーから作品の新規作成 / 名前変更 / 複製 / 削除
- 生成した物語は作品に紐付き、Theater で作品ごとに絞り込み表示
- Vault のカードはこれまでどおり全作品共通のアセット

### 2026-07-09 00:48 — Compose に生成設定を集約

- Compose 画面にプロット・目標トーン・清書プロンプトの指定と「生成する」ボタンを追加
- 「生成する」で Generate 画面に切り替わり自動で生成開始（進行表示に専念）
- 生成完了後は「Theater で再生 →」ボタンからそのまま鑑賞へ

### 2026-07-09 00:44 — 清書プロンプトのプリセット管理

- 物語の種類に合わせて複数の清書プロンプトを登録・切替できるように
  （設定画面で追加 / 編集 / 削除 / 切替。既定プロンプトは常にフォールバックとして残る）
- Generate 画面にプリセット選択ドロップダウンを追加
- 旧「単一上書き」からは自動移行

### 2026-07-09 00:30 — フェーズ 4: Compose（構成）+ 設定画面

- Compose: React Flow キャンバスでカードを配置し、線で繋いで並び順を決める
  （一本鎖になるよう接続を制約。「この構成を Generate へ」で生成画面に反映）
- 設定画面（ヘッダーの ⚙）を追加
- Theater: 本文のストリーミング表示（タイプライター演出）。設定でオン/オフ可能
- これで v1 の 4 フェーズ（Vault / Compose / Generate / Theater）が揃った

### 2026-07-09 00:15 — フェーズ 3: Theater（鑑賞）

- 物語の履歴一覧（再生 / 削除）
- 再生ビュー: Ken Burns（パン/ズーム）+ クロスフェード + テキスト長に応じたオート送り
- 動画カードはループ再生、メディア無しカードはグラデーション背景
- キーボード操作（スペース: 一時停止、← →: シーン移動、Esc: 終了）と終了画面

### 2026-07-08 23:51 — フェーズ 2: Generate（逐次生成）

- アンカー列を左から 1 シーンずつ清書し、確定事実（StoryState）を持ち越す生成パイプライン
- 生成は SSE でシーン毎に届き、画面にシーンが順に埋まっていく進行表示
- プロット・目標トーン（結末の着地）の指定
- 生成前に writer モデル（llama-server）を自動起動
- 清書用 system prompt の編集機能（上書き保存 / 既定に戻す。出力形式指示は自動付与）
- 物語は stories / story_scenes に保存（Theater で再生予定）

### 2026-07-08 23:14 — フェーズ 1: Vault（素材管理）

- シーンカードの登録・編集・削除（タイトル / ブリーフ / ロール / トーン / タグ 3 種）
- 画像・動画のアップロード（sha256 命名で `data/library/` に保存、サムネイル自動生成）
- キーワード検索（FTS5）と意味検索（ベクトル）、ロール絞り込み
- 在庫密度チップ（ロール別枚数）と埋め込み済み枚数の表示
- 類似カード確認（重複検知）
- 埋め込みサーバ（Qwen3-Embedding-4B）の自動起動とセットアップパネルでの起動/停止

### 2026-07-08 22:59 — フェーズ 0: アプリの足場

- Electron + React + TypeScript（electron-vite）+ FastAPI（venv）の骨格を実装
- 4 フェーズ（Vault / Compose / Generate / Theater）のタブ UI
- llama-server のアプリ内インストーラ（GitHub リリースから runtime/ へダウンロード・展開、
  進捗表示、キャンセル対応）と models/ 配下 GGUF のロード/停止 UI
- SQLite スキーマ（sqlite-vec / FTS5 含む）と Vault 在庫密度 API の初期実装
- `start.bat`（初回セットアップ + 起動）
