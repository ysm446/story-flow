# Changelog

## 未リリース

### 2026-07-12 — Compose / Generate: 作品・テイクの切り替え時に全体表示

- Compose で作品を開いたとき（起動時の復元・一覧からの切り替え・新規作成・複製・
  削除後の自動切り替え）、キャンバスを全ノードが収まる表示に自動調整するようにした
- Generate も同様に、テイクを表示したとき（起動時の最新テイク復元・一覧からの選択）に
  全体表示する。タブが非表示の間に読み込まれた場合は、タブを開いたときに調整される

### 2026-07-11 — 修正: 「使うフォルダ」未チェックでもおまかせが全フォルダから引いていた

- Compose の「使うフォルダ」を何もチェックしていないとき、アセット一覧はルート
  （共有）のカードだけを表示するのに、おまかせスロットの選定は全フォルダのカードを
  対象にしていた。チェックなし = ルートのみに統一し、アセット一覧に見えている
  カードとおまかせが選べるカードが常に一致するようにした

### 2026-07-11 — Compose: おまかせスロットをアセット一覧の先頭に

- ヘッダーの「＋ おまかせスロット」ボタンをやめ、アセット一覧の先頭に破線の
  「？」カードとして常設した。カードと同じ操作系（ドラッグでキャンバスに配置、
  クリックで右パネルに説明 + 配置ボタン）で扱える。配置しても消えず、何個でも置ける

### 2026-07-11 — Theater: 物語一覧にサムネイル

- 各物語の行の左に、メディア付きカードを使う最初のシーンのサムネイルを表示。
  メディアのない物語はグラデーションのプレースホルダになる

### 2026-07-11 — おまかせスロット: 連続する穴の選定を内分点方式に改善

- おまかせスロットが連続する場合（A〇〇〇B）、これまでは毎回 A と B の「中間」の
  ムードで候補を探していたため、どの穴もほぼ同じあたりのカードが候補になりがちだった
- 穴の位置に応じて検索点を A 寄り → B 寄りへ按分するようにした（1 つ目の穴は A 寄り、
  最後の穴は B 寄り）。序盤の穴には序盤らしいカード、終盤の穴には B へ繋がるカードが
  候補に出やすくなる。単独の穴の挙動は従来と同じ

### 2026-07-11 — 保存・更新の結果を下部ステータスバーに表示

- 自動保存には保存ボタンがなく「書いた文章が保存されたか」が分かりにくかったため、
  変更が確定したタイミングで下部ステータスバーに時刻付きで通知を出すようにした
  （数秒でフェードアウト。失敗時は赤く長めに表示）
- 対象: Compose の自動保存（プロット・追加指示・背景設定・構成の変更）、
  カードの作成/更新、BGM の作成/更新、プロンプトプリセットの保存
- あわせて、ステータスバー左端の backend 死活ドットとモデル名表示を撤去
  （モデル状態はヘッダーのモデルバーで確認できる）

### 2026-07-11 — Theater: シーンの待ち時間を指定できるように

- 設定に「シーンの待ち時間を指定」チェックボックスと秒数スライダー（0〜30 秒、既定 3.0）を追加。
  オンにすると、本文の表示（文字送り / オートスクロール）が終わってから指定秒数で
  次のシーンへ移る。オフのときは従来どおり本文の長さから自動で決める（既定）

### 2026-07-11 — Generate: 生成のキャンセルボタン

- 生成中（モデル起動中を含む）に左パネルからキャンセルできるように。
  中断するとサーバー側の清書も止まり、途中までの結果はテイクとして保存されない
  （画面には中断時点までの文章が残るが、別のテイクを表示すると消える）

### 2026-07-10 19:54 — Theater: 本文スクロール時に行が上下端で切れて見えないように

- 本文が長くスクロールする場合、これまでは表示枠の上端で行が途中から
  ばっさり切れて見えていた。上下端に 1 行ぶんのフェードを掛け、
  枠から出ていく行が自然に消えるようにした（本文フォントサイズに追従）

### 2026-07-10 06:42 — Theater: 最後のシーンをフェードアウトで終える

  そのあと終了画面（終）が静かにフェードインするように
  （従来は終了画面が即座に被さっていた）

### 2026-07-10 05:10 — Vault のフォルダ整理 + 作品ごとの「使うフォルダ」

- Vault のカードタブ左にフォルダツリーを追加。フォルダは入れ子にでき、
  カードをドラッグ&ドロップで出し入れ、フォルダ同士もドラッグで並べ替え・入れ子化できる
- 「ルート（共有）」のカードはどの作品でも常に使える共通素材。
  フォルダに入れたカードは、Compose の生成設定「使うフォルダ」で選んだ作品だけで使われる
  （アセット一覧とおまかせスロットの選定対象が絞られる。選択はサブフォルダを含む）
- 手置き済みのカードはフォルダ選択に関係なく常に有効
- フォルダの削除は「解体」: 中のカードとサブフォルダは 1 つ上の階層へ移動（カードは消えない）
- フォルダを開いた状態で新規カードを作ると、そのフォルダに入る

### 2026-07-10 03:51 — v1.5: おまかせスロット（パズル型穴埋め）

- Compose に「おまかせスロット」を追加（アセットエリアのボタンから配置）。
  カードの代わりに「？」ノードを鎖の中間に置くと、生成時に前後の流れに合うカードを
  在庫から LLM が 1 枚選んで埋める（候補検索 → LLM 選択の二段）
- おまかせスロットのプロパティで希望ロール・追加指示・BGM 指名を指定できる
- ロールは絞り込みではなく「近いカードが選ばれやすくなる」ボーナスとして働く
  （ロール未設定のカードは汎用として扱われる）
- Generate では選定の進行（カード選定中… → 選ばれたカードと選定理由）がノードに表示され、
  選定理由はテイクにも保存される
- 制約: 始点・終点にはカードが必要（おまかせは中間のみ）。部分再生成は確定済みの
  カード列に対して行う（穴の引き直しは「新しいテイクを生成」で）

### 2026-07-10 00:34 — LLM の思考モードを確実に無効化

- 思考モード（reasoning/thinking）対応モデルでも、清書・選曲が思考なしで実行されるように
  - サーバ起動引数に強制打ち切り（--reasoning-budget 0）と Qwen3 系向けの
    enable_thinking=false を追加
  - リクエスト側でも無効化を指定（手動起動したサーバに繋いだ場合にも効く）

### 2026-07-10 00:28 — 安定性の修正（プロジェクトレビューでの検出分）

- 修正: BGM をノードに指名したあとその BGM を削除すると、生成が全シーン完了した後の
  保存で失敗し、テイクが丸ごと失われていた（無効な参照は自動選曲/無音に劣化して保存）
- 修正: 生成中に別タブへ移動すると進行が見えなくなり、二重に生成を開始できてしまっていた
  （Compose / Generate は画面を切り替えても状態を保持するように）
- 修正: Compose 編集直後（0.8 秒以内）にタブを切り替えると、その編集が保存されず消えていた
- 修正: Compose でノードを追加・削除したあと、Generate の「新しいテイクを生成」が
  編集前の古い構成で生成されることがあった（常に現在のグラフから生成する）
- 修正: 生成が完了しないまま接続が切れた場合に成功と表示されていた（エラー表示に）
- 修正: カード保存（埋め込み計算）中に作品の自動保存などが重なると
  「database is locked」で失敗することがあった
- アプリの二重起動を禁止（2 つ目は既存ウィンドウを前面に出すだけ）
- 前回の異常終了で残った llama-server を起動時に自動回収
  （残っているとサーバが増殖して VRAM を余分に消費していた）
- モデルの起動失敗（ファイル破損など）を 90 秒待たずに即エラー表示
