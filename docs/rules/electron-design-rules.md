# Electron デザインルール

作成日時: 2026-06-11 00:00
更新日時: 2026-06-11 00:00

この文書は、Electron + React 系のデスクトップアプリで再利用するための UI デザインルールです。
LM Chat 固有の実装に閉じず、他プロジェクトでも参照できる基準としてまとめます。

## 基本方針

- デスクトップアプリは長時間使われる前提で、情報密度、視認性、疲れにくさを優先する。
- 見た目の装飾より、操作対象の位置、サイズ、状態がすぐ分かることを優先する。
- サイドバー、エディタ、設定画面、一覧、詳細ペインは、同じ余白・フォント・区切りの感覚で統一する。
- OS ネイティブ UI に近い控えめな質感を基本にし、過度な角丸、強いグラデーション、大きな影は避ける。
- hover / active / focus / disabled / loading の状態を必ず設計する。
- ドラッグ、リサイズ、スクロールなどの操作領域は、見た目より少し広く取る。

## フォント

### フォントファミリー

- UI 全体は system font stack を基本にする。
- 日本語を含むアプリでは、Windows では `Yu Gothic UI`、macOS では `Hiragino Sans`、共通 fallback として `Noto Sans JP` を想定する。
- 長文エディタ、チャット本文、ドキュメント本文は、UI ラベルより読みやすさを優先して `system-ui` ベースまたは設定可能な本文フォントにする。
- コード、パス、モデル名、ログ、数値 ID は monospace を使う。

推奨例:

```css
font-family:
  system-ui,
  -apple-system,
  BlinkMacSystemFont,
  "Segoe UI",
  "Yu Gothic UI",
  "Hiragino Sans",
  "Noto Sans JP",
  sans-serif;
```

### フォントサイズ

- 最小本文サイズは `13px` とする。常用 UI で `12px` 未満を使わない。
- 通常の UI ラベル、ボタン、一覧項目は `13px` から `14px` を基本にする。
- チャット本文、ドキュメント本文、長文編集欄は `15px` から `16px` を基本にする。
- 設定値、補助説明、メタ情報は `12px` から `13px` に留める。
- 見出しは階層を増やしすぎず、`16px`、`18px`、`20px` 程度で抑える。

### 行間

- UI ラベルや一覧項目は `1.35` から `1.45`。
- チャット本文やエディタ本文は `1.55` から `1.7`。
- ボタンやツールバー内の一行テキストは、固定高の中央揃えで読みやすくする。

## 色とコントラスト

- 背景、パネル、サイドバー、入力欄、hover の階層差は小さくても明確にする。
- 境界線は強い線ではなく、背景差と薄い border を組み合わせる。
- テキスト色は、本文、補助、placeholder、disabled の 4 段階を用意する。
- accent color は操作対象と現在位置の強調に限定し、画面全体を単一色で染めない。
- ダークテーマでは、真っ黒背景と真っ白本文を避け、少し明度を寄せて目の負担を減らす。

## 余白と密度

- 基本単位は `4px` または `8px` にそろえる。
- ツールバー、一覧、設定項目など高密度 UI では `4px` / `6px` / `8px` を中心に使う。
- パネル内のまとまりは `12px` から `16px`、大きなセクション間は `20px` から `24px`。
- デスクトップアプリではランディングページ的な広すぎる余白を避ける。
- 情報量が多い画面では、余白より alignment とグルーピングで読みやすさを作る。

## 角丸と影

- 通常のボタン、入力欄、リスト項目、カードの角丸は `4px` から `8px`。
- 小さなアイコンボタンは `4px` から `6px`。
- モーダルやポップオーバーも `8px` 程度に抑える。
- 影は浮いた要素の階層表現に限定し、常時表示のパネルやカードには多用しない。
- サイドバーや分割ペインは shadow より border / background で区切る。

## ボタン

- アイコンで意味が通る操作はアイコンボタンを使い、必要に応じて tooltip を付ける。
- 主要アクションだけ text button または icon + text button にする。
- 小さな toolbar button の推奨サイズは `28px` から `32px`。
- 通常ボタンの推奨高さは `30px` から `36px`。
- 危険操作は色だけに頼らず、確認 dialog、文言、配置で区別する。
- disabled は opacity だけでなく cursor と hover 無効化も合わせる。

## 入力欄

- 一行 input の高さは `30px` から `36px`。
- textarea / editor は本文フォントと行間を優先し、上下 padding を十分に取る。
- placeholder は薄くしすぎず、本文との差が分かる程度にする。
- focus ring は必ず出す。border color だけでなく、必要なら outline / box-shadow を併用する。
- 入力中に高さが変わる UI は、最大高とスクロール挙動を決めてレイアウトジャンプを抑える。

## スライダー

- スライダーは目立ちすぎない細さにする。
- track の推奨高さは `2px` から `4px`。
- thumb の推奨サイズは `12px` から `16px`。
- 操作しやすさのため、見た目の track より広い hit area を持たせる。推奨 hit area は高さ `24px` 以上。
- thumb は hover / active / focus で少しだけ強調する。
- 数値を厳密に調整する設定では、スライダーだけでなく数値 input、reset button、現在値表示を併用する。
- デフォルト値から変更されている状態は、accent color や indicator で分かるようにする。

CSS の考え方:

```css
.slider {
  height: 24px;
}

.slider-track {
  height: 3px;
}

.slider-thumb {
  width: 14px;
  height: 14px;
}
```

## サイドバー

- サイドバーは、ナビゲーション、ワークスペース、一覧、補助操作を置く場所として使う。
- 左サイドバーは現在位置と対象選択を主目的にする。
- 右サイドバーは詳細設定、補助情報、文脈操作など、主作業を邪魔しない情報に使う。
- サイドバー幅は固定ではなく、リサイズ可能にする。ただし最小幅と最大幅を必ず決める。
- 推奨最小幅は `220px`、推奨標準幅は `280px` から `340px`、推奨最大幅は画面幅の `40%` 程度。
- 折りたたみ状態では、主要な復帰操作が分かるように icon / rail / toggle を残す。
- サイドバー内のリスト項目は、高さ `28px` から `36px` を基準にする。
- 選択中、hover、ドラッグ中、ドロップ先の状態を明確に分ける。

## 分割ペインとリサイズハンドル

- 見た目の divider は `1px` でも、操作可能な hit area は広く取る。
- リサイズハンドルの推奨 hit area は `8px` から `12px`。
- 視覚上の線は中央に `1px` から `2px` で描く。
- hover 時に cursor を `col-resize` / `row-resize` に変える。
- hover / dragging 中は divider の色や背景を少し強調する。
- リサイズ中は text selection を抑制し、意図しない文字選択を防ぐ。
- ダブルクリックで標準幅に戻す挙動を用意できると扱いやすい。

CSS の考え方:

```css
.resize-handle {
  width: 10px;
  cursor: col-resize;
}

.resize-handle::before {
  width: 1px;
}
```

## スクロールバー

- Electron ではスクロール領域が多くなるため、scrollbar は細く控えめにする。
- 推奨幅は `8px` から `10px`。
- thumb は通常時控えめ、hover 時に少し濃くする。
- track は背景になじませ、常に強い枠として見せない。
- エディタ、チャット、サイドバーのスクロールバーは太さと色を統一する。

## リストとツリー

- 一覧項目はタイトル、補助情報、操作ボタンの優先度を明確にする。
- 右端の操作ボタンは hover 時にだけ表示してもよいが、重要操作は discoverable にする。
- ドラッグ可能な項目は、ドラッグ中の ghost、ドロップ先、禁止状態を用意する。
- 長いタイトルは 1 行省略を基本にし、必要なら tooltip または詳細ペインで全文を見せる。
- delete / rename / duplicate / move などは同じ位置・同じメニュー構造にそろえる。

## タブとセグメント

- タブは画面内の大きな表示切り替えに使う。
- セグメントコントロールは同じ設定軸の切り替えに使う。
- active 状態は背景、border、文字色の組み合わせで明確にする。
- タブ同士の高さ、padding、border radius を統一する。
- タブの文字が折り返されるほど狭くなる場合は、横スクロールまたは icon 化を検討する。

## モーダルとポップオーバー

- モーダルは破壊的操作、重要設定、複雑な選択に限定する。
- 軽い選択や補助操作は popover / menu / inline panel にする。
- モーダルの幅は内容に合わせる。設定フォームは `420px` から `640px` 程度を基本にする。
- close、cancel、confirm の位置を一貫させる。
- Escape、外側クリック、Enter の扱いを画面ごとに明確にする。

## エディタとチャット本文

- 長文入力は読みやすさを最優先し、本文幅、行間、フォントサイズを調整可能にする。
- チャット本文は `max-width` を持たせ、横に伸びすぎないようにする。
- 編集中 textarea は元の本文位置と近い見た目にして、編集モードへ移ったことが自然に分かるようにする。
- Markdown、コード、引用、リストの表示は本文より少しだけ階層差を出す。
- コードブロックは monospace、背景、border、copy button の状態をそろえる。

## アイコン

- アイコンは意味が一般的なものを使う。独自アイコンを増やしすぎない。
- 推奨サイズは `16px` から `18px`。
- 小さなボタン内では `16px`、主要 toolbar では `18px`。
- アイコン単体ボタンには tooltip または aria-label を付ける。
- active / disabled / dangerous の色変化はテキストボタンと同じルールにする。

## アクセシビリティ

- キーボードだけで主要操作ができるようにする。
- focus visible を消さない。
- icon button には `aria-label` を付ける。
- 色だけで状態を伝えない。
- clickable な領域は最低 `24px` 以上、頻繁に押す操作は `28px` 以上を確保する。
- テキストは十分なコントラストを持たせ、disabled でも読めないほど薄くしない。

## 実装時のチェックリスト

- [ ] フォントサイズが小さすぎない。
- [ ] スライダーの track は細く、hit area は十分に広い。
- [ ] thumb は `12px` から `16px` 程度で、hover / active / focus がある。
- [ ] サイドバー幅に最小・標準・最大の基準がある。
- [ ] divider の見た目は細く、操作領域は `8px` から `12px` ある。
- [ ] hover、active、focus、disabled、loading の状態がある。
- [ ] スクロールバーの太さと色が画面間で統一されている。
- [ ] 長いテキストが UI を押し広げたり重なったりしない。
- [ ] アイコンボタンに tooltip または aria-label がある。
- [ ] 危険操作に確認や取り消し手段がある。
