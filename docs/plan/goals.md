# goals.md — プロジェクトの目的と完成形

作成日時: 2026-07-08 16:39
更新日時: 2026-07-08 16:39

## 何を作るか

**story-flow** — 短編ストーリー生成・鑑賞デスクトップアプリ。

作者が事前に「シーンカード」（メディア + 短い指示文 = ブリーフ）を大量に用意しておき、
始点・終点（と任意の中間点）を置くと、ローカル LLM が在庫カードから間を埋め、
左から 1 シーンずつ清書して一本の物語に仕立て、それを鑑賞する。
`lm-graph` をフォークして構築する。

詳細仕様は [docs/spec.md](../spec.md) が唯一の正。本ファイルはその要約と価値基準。

## 面白さの核（最重要価値）

- **リプレイ性**: 同じ素材から毎回すこしずつ違う短編が生まれること。これが生命線。
- カードは完成した本文ではなく**作者の意図（ブリーフ）**。清書は全編 LLM が書き、
  読み味が毎回変わる。ただしブリーフの意図・確定事実は壊さない。
- 一貫性: 逐次生成 + StoryState（確定事実）の持ち越しで「3 枚目の赤い傘が
  5 枚目で消える」型のドリフトを防ぐ。

## 完成形

### v1（まずここを完成させる）

- **Vault**: カード CRUD、メディア登録、タグ/ロール、埋め込み計算、検索、在庫密度可視化 — フル実装
- **Compose**: React Flow で始点・中間・終点アンカーを置いて繋ぐ（順序 = 並びのみ）
- **Generate**: 逐次清書 + StoryState 持ち越し（穴埋めなし、FIXED スロットのみ）
- **Theater**: Ken Burns + オート送り + クロスフェードで鑑賞

### v1.5（本命のパズル型）

- `fill_gap`: 候補検索（sqlite-vec + FTS + ロール）→ LLM 選択の二段で穴を自動で埋める
- 多様性制御、浅いバックトラック、target_tone の引力、在庫密度チューニング

### v2（構想）

- 引力（アトラクタ）方式の分岐・複数終端、鑑賞後の軌跡マップ、TTS + BGM、VN 的選択分岐

## 重視する価値・判断基準

1. 迷ったら「一本道 → 逐次清書 → 鑑賞」の最小経路を優先し、多機能化しない。
2. spec.md の「### 判断」ブロックは確定事項。実装時に蒸し返さない。
3. ML コンポーネントは再実装せず、subprocess / HTTP でラップする。
4. データの流れは `Vault → Compose → Generate → Theater` の一方向。
   清書結果は保存するが index しない（`DB → 物語` の一方向、逆流させない）。

## 非目標

- 一括生成（全シーンをまとめて LLM に書かせること）は採らない。
- v1 の段階で v1.5 / v2 の機能を実装しない（差し込める設計にはする）。
- 完全なビームサーチ等の重い探索は v1.5 でも実装しない。
