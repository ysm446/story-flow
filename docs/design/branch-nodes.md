## アイデア

シーンの途中に**選択肢**を置き、鑑賞時にプレイヤーが枝を選ぶとゲーム（ADV）っぽくなる。
Compose で 1 つのノードから出力エッジを 2 本以上引けるようにし、各エッジ = 選択肢とする。

## 実現性（2026-07-11 評価: 中〜大。ワークスペース導入級だが作り直しではない）

アーキテクチャは分岐を先取りしている:

- **state_after スナップショット**: 各シーンが終了時点の確定事実（StoryState）を保存済みで、
  部分再生成はこれを復元して途中から書き直している。分岐 = 「同じスナップショットから
  2 本の続きを書く」なので、生成エンジンはこの既存機構の延長で作れる。
- **逐次生成の原則は維持できる**: 枝ごとに左から順に清書すれば「生成は逐次のみ」に抵触しない。
- **Compose は React Flow**: 出力エッジ複数はネイティブに描ける。制約検証を
  「一本鎖」→「木」に変えるだけ（循環禁止は維持）。

手間が掛かる 3 箇所:

1. **物語の保存形式**: story_scenes は position の一直線。木構造への
   スキーマ変更 + 移行が最大の設計ポイント（シーンに parent を持たせる、または
   経路ごとに保存して共通プレフィックスを共有）。
2. **Theater**: 線形インデックス送り → 「分岐点で選択肢を出して待つ → 選んだ枝へ進む」。
   選択肢 UI・BGM/トランジションの追従を含めボリュームあり（ADV らしさの本体）。
3. **選択肢テキストの出どころ**: まずは**エッジのプロパティとして作者が書く**のがシンプル。
   「枝の先のブリーフから LLM に選択肢文を書かせる」は後から追加できる。

## 確定しておく制約（最小形）

- **木のみ。合流（分かれた枝が 1 本に戻る）はやらない** — 枝ごとに StoryState が
  違ってしまい、合流後のシーンがどちらかの枝と矛盾するため。
- 生成コストは枝の分だけ線形に増える（10 シーン中盤に 2 択 1 つなら共通 5 + 5×2 = 15 シーン分）。
  分岐点が増えると掛け算で伸びるため、逐次生成が快適な速度であることが前提。
- おまかせスロットは枝の中でも使える想定（fill_gap は枝単位の左→右で動く）。
